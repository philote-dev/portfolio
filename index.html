FRC Simulations

World Championship Competition System

Overview

Comprehensive robotics system developed for FIRST Robotics Competition, culminating in a semifinal appearance at the 2023 World Championship Galileo Division Finals. This suite represents a year of intensive development combining simulation, computer vision, and autonomous systems.

Key Components

  • Physics-Based Simulation: Custom simulator for driver practice and software testing without physical robot access, reducing development time and hardware wear
  • Autonomous Object Avoidance: Real-time path planning system using OAK-D Pro camera with dynamic obstacle detection and path stitching
  • AI Object Detection: Integrated computer vision pipeline for game piece recognition and field element tracking
  • Robot State Machine: Robust state management architecture ensuring reliable autonomous and teleoperated modes

Technical Implementation

Built primarily in Java with Python integration for computer vision tasks. The object avoidance system uses PathPlanner paths with real-time nulled zones based on OAK-D Pro camera data, enabling dynamic replanning during autonomous periods.

NovelGym HITL Extension

RSI 2024 Research Project | MuLIP Labs, Tufts University

Research Overview

Extended the NovelGym framework to incorporate Human-in-the-Loop (HITL) feedback mechanisms for reinforcement learning agents operating in open-world scenarios with novelty. This research addresses a critical challenge in AI: how can agents adapt to unexpected situations by leveraging human expertise?

Key Contributions

  • HITL Framework: Developed interface for real-time human feedback integration during agent training and deployment
  • Feedback Processing: Created mechanisms to translate natural language human guidance into actionable agent updates
  • Adaptability Metrics: Established quantitative measures for agent improvement with human assistance
  • GridWorld Implementation: Demonstrated system in controllable environment with diverse novelty types

Technical Approach

Built on Python-based NovelGym infrastructure using PDDL for domain specification and reinforcement learning algorithms for agent training. The HITL system accepts human feedback through multiple modalities and updates agent policies without full retraining.

Results & Impact

Demonstrated significant improvement in agent adaptability when human feedback was incorporated, particularly in scenarios with unexpected environmental changes. This work contributes to the broader goal of creating AI systems that can collaborate effectively with humans in uncertain environments.

Rubik's Cube Solver

Custom Kociemba Algorithm Implementation

Project Motivation

After years of speedcubing (averaging ~10 second solves), I wanted to understand the mathematics behind optimal solving algorithms rather than just memorizing patterns. This project became a deep dive into heuristic search, constraint satisfaction, and mechanical engineering.

Algorithm Development

  • Phase-State Decomposition: Implemented two-phase approach separating cube into subsets for efficient solving
  • Pruning Tables: Generated comprehensive lookup tables to eliminate impossible move sequences
  • Optimization: Custom heuristics reducing average move count and computation time
  • Search Strategy: Iterative deepening A* with domain-specific pruning

Hardware Implementation

Designed and 3D-printed mechanical solver on Bambu Lab P1S. Six servo-controlled arms manipulate the cube with coordinated timing. Integration between software solver and hardware controller required careful calibration and error handling.

Performance

Achieves solve times of 21-41 seconds with move counts of 19-31, balancing optimality with computational efficiency. The project taught me invaluable lessons about the gap between theoretical algorithms and practical implementation.

Key Learnings

This project embodied patience and iteration—debugging mechanical timing issues, optimizing search algorithms, and calibrating servo movements required persistence. It reinforced my belief that the best learning comes from tackling problems just beyond your current skill set.

AI Sketch Visualizer

Active Development | AI@MIT

Project Vision

Creating an AI-powered engineering tool that democratizes design validation by converting hand-drawn sketches into 3D models with integrated manufacturability analysis. This addresses a critical barrier: students designing components for projects often lack the machining background to assess feasibility before investing in expensive CAD work.

Technical Approach

  • Computer Vision Pipeline: Processes isometric or multi-view orthographic sketches, cleaning and extracting geometric features
  • 3D Model Generation: Converts 2D sketches into 3D meshes using CV model trained on engineering drawings
  • Manufacturability Analysis: Evaluates generated models for undercuts, overhangs, minimum feature sizes, and other machining constraints
  • LLM Integration: Provides natural language feedback on design feasibility and suggested modifications

Current Status

In active development phase—prototyping CV model for sketch interpretation and establishing analysis framework. Key challenges include handling varied sketch quality, ambiguous geometric specifications, and balancing analysis depth with user accessibility.

Planned Features

  • Interactive UI for sketch input and model visualization
  • Color-coded feasibility mapping (green/yellow/red highlighting)
  • Complexity scoring indicating required machining capabilities
  • Alternative design suggestions for problematic features

Future Impact

This tool aims to lower barriers to hardware prototyping, especially for students without access to premium CAD software or machining expertise. By providing rapid feedback on design feasibility, it enables faster iteration and more ambitious projects.

iQuHack 2026 — Quantum Rings Challenge

Quantum Circuit Difficulty & Runtime Prediction

Overview

Team-based competition project at iQuHack (Winter 2026) working with Ethan Herman, Hazel Torek, Ryon Peddapalli, and Varun Sethi. The challenge required predicting minimum fidelity threshold and runtime of quantum circuits using static QASM-only features—no simulation-dependent features allowed. This constraint forced us to develop novel structural analysis techniques that could extract circuit difficulty from topology alone.

Key Components

  • QASM Static Feature Extraction Pipeline: Designed comprehensive feature extraction system analyzing circuit structure without execution
  • 2-Qubit Interaction Graph: Built graph representation modeling entanglement structure and connectivity patterns
  • Structural Metrics: Derived max qubit degree, degree variance, two-qubit gate density, entanglement volume, and temporal packing metrics
  • Entanglement Progression Compound Metric: Introduced novel compound metric capturing how entanglement complexity evolves through circuit layers
  • Fidelity Curve Modeling: Modeled fidelity curve using logistic regression fitting, extracting phase-shift parameter representing intrinsic circuit difficulty
  • Multi-Stage Threshold Prediction: Built system combining linear regression baseline, random forest model, and feature-aware binning correction
  • Runtime Prediction Neural Network: Implemented PyTorch network with 12 feature input, Dense(128) → ReLU → Dense(64) → ReLU → Output log2(seconds) architecture

Technical Implementation

The core insight driving our approach was that circuit difficulty is multiplicative, not additive. Compound structural features significantly outperformed individual features. We validated our models across GHZ, QFT, Grover, and QAOA circuit families, demonstrating strong correlation between static features and both fidelity thresholds and runtime. The neural network regression model for runtime prediction operated on log2-transformed seconds to handle the wide dynamic range of execution times.

Demonstrated reliable threshold prediction using purely static analysis features, improving generalizability without compilation-dependent metrics. This work shows that structural circuit analysis can provide meaningful predictions about quantum circuit behavior without expensive simulation, enabling faster circuit design iteration and resource allocation in quantum computing systems.

MNIC — AI Mnemonic Generator

AI-Powered Musical Mnemonics (AI@MIT Winter 2025)

Overview

Developed in collaboration with Ajibola Ajani and Angelo Farfan at AI@MIT (Winter 2025). The project addresses a fundamental learning challenge: students retain melodic information better than plain text, yet no unified tool exists to automatically generate structured musical mnemonics with vocals and harmony. MNIC bridges this gap with an end-to-end AI and DSP pipeline transforming structured knowledge into melody-driven memory reinforcement.

Key Components

  • Lyrics Generation: Claude Sonnet 4 API integration supporting rhythmic rhyme schemes (AABB, ABAB) and acrostic mnemonics, outputting mnemonic.txt with metadata JSON
  • Vocal Generation: Suno Bark TTS producing 24kHz WAV output with syllabic mapping and prompt conditioning for natural speech rhythm
  • Audio Feature Extraction: librosa-based pipeline using pyin() pitch detection, beat_track(), and chroma_cqt() for key and tempo extraction
  • Harmony Generation: I–IV–V–I chord progression generation using pretty_midi with FluidSynth rendering for musical accompaniment
  • Audio Stitching: Numpy-based mixing pipeline with Muspy integration, normalization, and 44.1kHz final output production

Technical Implementation

The pipeline processes input knowledge structures through sequential stages: first generating semantically appropriate lyrics with proper rhythmic structure, then synthesizing natural-sounding vocals, extracting musical features from the vocal track, generating harmonically compatible chord progressions, and finally mixing all components into a cohesive musical mnemonic. Performance averages 30–45 seconds per generation on Apple Silicon (M4), making it practical for real-time educational use.

FRCxMIT Ri3D 2026

72-Hour Rapid Robotics Prototyping

Overview

Collaborated with FRCxMIT to design, build, and program a fully functional FIRST Robotics robot within 72 hours of game release. The Robot in 3 Days (Ri3D) challenge tests teams' ability to rapidly prototype, integrate, and deploy complex robotic systems under extreme time constraints. This project demanded architectural decisions that balanced functionality with speed of implementation, real-time debugging discipline, and modular design enabling parallel development.

Key Components

  • Rapid Autonomous Architecture: Designed autonomous system architecture optimized for quick deployment under extreme time constraints
  • Modular Java Command-Based Structure: Implemented WPILib command-based robot framework enabling parallel development and rapid iteration
  • Vision and Field-Relative Control: Integrated computer vision pipeline with field-relative positioning for accurate autonomous navigation
  • Drivetrain and Mechanism Control Loops: Tuned PID controllers and feedforward systems for precise motion control across multiple subsystems
  • Rapid Iteration Testing Pipeline: Established testing methodology enabling quick validation cycles without compromising system reliability

Technical Implementation

Built on Java with WPILib framework, focusing on speed of deployment, architectural cleanliness, and real-time debugging under competition conditions. The modular command-based structure allowed team members to work simultaneously on different subsystems while maintaining system integration. Vision integration required careful calibration and field-relative coordinate transformations to ensure autonomous path planning accuracy. Control loop tuning balanced responsiveness with stability, critical for reliable performance in competition scenarios.

Focus Areas

The project emphasized speed of deployment, architectural cleanliness, and real-time debugging discipline. Every design decision prioritized rapid iteration—choosing proven patterns over experimental approaches, maintaining clear interfaces between subsystems, and establishing debugging workflows that could identify and resolve issues within minutes rather than hours.

Impact

Demonstrated rapid system integration, real-time debugging discipline, and deployable autonomy under strict 72-hour constraints. This experience reinforced the importance of architectural decisions made early in development, the value of modular design for parallel work, and the critical role of disciplined testing practices when time is the primary constraint. The project showcases how robust engineering practices can be maintained even under extreme time pressure.
